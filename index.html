How It Works

Four steps.
Full picture.

1

Ingest Spans

Your tools send OTLP/HTTP spans to Specta on port 4318. Claude Code, MCP servers, and custom apps all speak OTLP natively.

OTLP/HTTP · port 4318
2

Store in DuckDB

Every span is stored in embedded DuckDB with AI-specific schema — tokens, cost, model, user, session, tool calls. No external database.

embedded DuckDB · Parquet export
3

Agents Analyze

10 specialized agents run continuously — detecting anti-patterns, correlating anomalies, building forecasts, generating per-developer coaching insights.

10 agents · Claude Sonnet 4.6
4

Alert & Coach

Alerts fire on budget overruns, error spikes, and anti-patterns. Coaching surfaces in dashboards. Reports are live — no batch jobs, no lag.

real-time · no ETL pipeline

Integration

One endpoint.
Complete observability.

Point your tools at Specta's OTLP endpoint. Everything else is automatic — agents start, dashboards populate, alerts configure themselves.

claude_code_config.json
{
  "OTEL_EXPORTER_OTLP_ENDPOINT":
    "http://localhost:4318",
  "OTEL_SERVICE_NAME": "claude-code",
  "OTEL_TRACES_EXPORTER": "otlp",
  "OTEL_METRICS_EXPORTER": "otlp"
}
Terminal
# Download and run Specta
java -jar specta.jar

# Dashboard opens at :8080
open http://localhost:8080

# Verify spans are arriving
curl -s http://localhost:8080/api/health
# {"status":"UP","spans_today":1240}
1

Run Specta anywhere

3

Agents activate automatically

Within minutes of first spans arriving, all 10 agents start analyzing. Anti-patterns surface, flow states appear, cost forecasts calibrate to your team's actual usage.

4

Configure alerts and share dashboards

Set budget thresholds, anti-pattern sensitivity, and latency SLOs. Share role-specific views via 7 built-in persona dashboards — from CFO to SRE to individual developer.
