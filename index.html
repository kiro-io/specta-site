How It Works

Four steps.
Full picture.

1

Ingest Spans

Your tools send OTLP/HTTP spans to Specta on port 4318. Claude Code, MCP servers, and custom apps all speak OTLP out of the box.

OTLP/HTTP · port 4318
2

Store in DuckDB

Every span is stored in embedded DuckDB with AI-specific schema — tokens, cost, model, user, session, tool calls. No external database needed.

embedded DuckDB · Parquet export
3

Agents Analyze

10 specialized agents run continuously — detecting anti-patterns, correlating anomalies, building forecasts, and generating coaching insights per developer.

10 agents · Claude Sonnet 4.6
4

Alert & Coach

Alerts fire on budget overruns, error spikes, and anti-patterns. Developers receive coaching directly in their sessions. Reports land in dashboards in real time.

p99 latency · cost · acceptance rate

Integration

Complete observability.

Point your tools at Specta's OTLP endpoint. Everything else is automatic.

{

}

# 1. Download and run Specta

1

Run Specta locally or on-prem

Single java -jar specta.jar. No Docker required, no external database, no cloud account. Listens on :4318 for spans and :8080 for the dashboard.

2

Point your tools at the OTLP endpoint

Set OTEL_EXPORTER_OTLP_ENDPOINT=http://localhost:4318 in your Claude Code config, MCP servers, or any OTEL-instrumented app. Specta speaks standard OTLP/HTTP.

3

Specta activates its 10 agents

Within minutes of first spans arriving, agents start analyzing — anti-patterns surface, flow states appear, cost forecasts begin calibrating to your team's actual usage.

4

Set alert rules and share dashboards

Create budget alerts, anti-pattern thresholds, and latency SLOs. Share per-developer or per-team views via role-based dashboard links — 7 persona views built in.
